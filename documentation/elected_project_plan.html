🗳️ ElectED 2.0 — Full-Scale Competition Submission
An intelligent, civic education platform. Not just a chatbot — a living civic companion with Google integration, real election data, personalized voter journeys, and community features.
React + TypeScript Google Gemini API Firebase / Firestore Unique among submissions 6-day delivery plan
What sets this apart
🧭
Personalized voter journey
User answers 3 questions (state, election type, first-time?). App generates a custom, step-by-step action plan for their specific situation. No one else will do this.
📡
Live election data
Pulls real polling place data, registration deadlines, and upcoming races using Google Civic Information API. Dynamic, not static.
🗣️
Multi-turn AI with memory
Gemini-powered assistant remembers context across the session, detects confusion, and proactively suggests related topics. Feels like a conversation, not a search.
📊
Visual explainers
Interactive animated diagrams for complex topics: how Electoral College math works, vote counting flow, redistricting impact. No competitor will have these.
📅
Google Calendar integration
One-click export of personal election deadlines to Google Calendar with smart reminders. A deeply useful, evaluator-visible Google Service integration.
🌍
Multilingual civic access
Google Translate API integration makes content accessible in 10+ languages. Directly addresses accessibility and inclusion — an evaluation criterion.

6-day delivery plan
Day 1
Foundation & Architecture
Scaffold React + TypeScript + Vite project, configure Tailwind and ESLint
Set up React Router with all 6 routes stubbed out
Create Firebase project, enable Firestore + Hosting
Build design system: tokens, typography, color variables, reusable components (Button, Card, Badge)
Build sidebar nav + mobile header + responsive layout shell
Push initial skeleton to GitHub with CI setup
✅ End of day: Working skeleton deploys to Firebase Hosting. All routes navigate. Design system in place.
Day 2
Onboarding + Dashboard + AI Chat
Build onboarding quiz (state selector, election type, experience level) — stores to Firestore
Build personalized dashboard: dynamic checklist based on user state, deadline countdown cards
Integrate Google Gemini API via Firebase Function proxy (keeps API key secure)
Build full AI chat UI: message bubbles, typing indicator, quick-chip prompts, conversation history
Write and test nonpartisan system prompt, verify refusal of partisan questions
✅ End of day: Users can onboard, see a personalized dashboard, and have a full AI conversation.
Day 3
Google Civic API + Maps + Calendar
Integrate Google Civic Information API: fetch polling places and election info by address
Build Voter Tools page: address input → Maps embed showing polling locations
Implement Google Calendar integration: generate .ics file + direct "Add to Google Calendar" deeplink for each deadline
Add Translate API toggle in Settings: translate key UI strings to 10 supported languages
Handle API error states and loading states gracefully throughout
✅ End of day: All Google API integrations working. Polling finder live. Calendar export works. Language toggle works.
Day 4
Education Hub — Timeline, Quiz, Glossary + Visual Explainers
Build animated Election Timeline (Framer Motion scroll-triggered reveal)
Build adaptive quiz engine: 15 questions, difficulty scoring, explanations, score persistence in Firestore
Build searchable Glossary with 35+ terms, category filters, related term linking
Build 2 interactive SVG visual explainers: Electoral College vote calculator + vote counting process flow
Build 8-step Voting Guide cards with state-specific notes
✅ End of day: Full Education Hub complete. All 5 sections working. Interactive visualizations rendering.
Day 5
Polish, Accessibility, Testing + Security
Accessibility audit: keyboard navigation, ARIA labels, color contrast (WCAG AA), screen reader test
Security: move all API keys to Firebase env vars, add input sanitization, validate all user input
Write unit tests with Vitest: AI chat utility functions, quiz engine, data helpers — target 80% coverage on core logic
Mobile responsive QA: test every view at 375px, 768px, 1280px
Performance: lazy-load routes, compress assets, verify Lighthouse score ≥ 85
Animation polish pass: stagger reveals, micro-interactions on hover
✅ End of day: App is polished, tested, accessible, and secure. Lighthouse 85+. All known bugs fixed.
